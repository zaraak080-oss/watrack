# Deployment Guide: WA Panel Vault

This guide explains how to host your WhatsApp Monitoring Dashboard on a server or bot hosting provider.

---

## 1. What to Upload
When uploading the project to your server (via FTP, SFTP, GitHub, or file manager), upload the following files and folders:

### ✅ Upload:
* `src/` (Source files for frontend)
* `index.js` (Backend entrypoint)
* `package.json` & `package-lock.json` (Dependencies configuration)
* `vite.config.ts` & `tsconfig.json` & `tsconfig.app.json` & `tsconfig.node.json` (Build configurations)
* `index.html` (Frontend HTML template)

### ❌ DO NOT Upload (Ignore/Delete before uploading):
* `node_modules/` (Will be installed fresh on the server)
* `dist/` (Will be built on the server or generated by build commands)
* `sessions/` (Contains your local auth keys. A new folder will be created on the server when pairing)
* `wa-panel-vault/` (Contains local message logs. A new folder will be created on the server)

---

## 2. Essential Folders to Persist (Critical)
The application stores persistent data on the local filesystem. If you are using a hosting provider with **ephemeral storage** (e.g. Render, Heroku, or Docker-based hosts), you **MUST** map these folders to **Persistent Volumes**:

1. **`sessions/`**: Stores Baileys authentication states. If lost, your WhatsApp session will disconnect, and you will need to re-pair via QR code or Pairing Code.
2. **`wa-panel-vault/`**: Stores all text-based message logs and downloaded media files.

---

## 3. Environment Variables
You can configure the server using environment variables:

| Variable | Description | Default |
| :--- | :--- | :--- |
| `PORT` | The port the web server runs on | `3000` |
| `WEBHOOK_URL` | Optional URL to send real-time message JSON payloads | *None* |

---

## 4. Webhook Notification Payload
If `WEBHOOK_URL` is set, the server sends a POST request with JSON payload for every received message:

### Text Message:
```json
{
  "event": "text_message",
  "direction": "INCOMING",
  "isGroup": false,
  "sessionName": "my-device",
  "contactName": "John_Doe",
  "content": "Hello, how are you?",
  "timestamp": "14:23:45 02/07/2026"
}
```

### Media Message:
```json
{
  "event": "media_message",
  "direction": "INCOMING",
  "isGroup": false,
  "sessionName": "my-device",
  "contactName": "John_Doe",
  "mediaUrl": "/vault/my-device/media/1719912039_abc12.jpg",
  "timestamp": "14:25:10 02/07/2026"
}
```

---

## 5. Deployment Commands

Run the following commands in the project folder on your server:

```bash
# 1. Install dependencies
npm install

# 2. Build the frontend assets
npm run build

# 3. Start the production server
npm start
```

---

## 6. Keeping the Server Running (PM2 Process Manager)
To ensure the server runs continuously in the background and restarts automatically on crashes or server reboots, it is highly recommended to use **PM2**:

1. Install PM2 globally:
   ```bash
   npm install -g pm2
   ```

2. Start the application:
   ```bash
   pm2 start index.js --name "wa-panel-vault"
   ```

3. Setup PM2 to run on system boot:
   ```bash
   pm2 startup
   pm2 save
   ```
